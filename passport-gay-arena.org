* Preamble: The Signed Trit as Mantissa-Significand

The interval [-1, 1] is the *treacherous indeterminism interval* of FFI choices.

Every language represents signed integers differently at the FFI boundary:
- Go: =int8= (two's complement, -1 = 0xFF)
- Clojure/JVM: =long= (two's complement, -1 = 0xFFFFFFFFFFFFFFFF)
- Julia: =Int8= (two's complement)
- Move: *no signed integers at all* — only =u8=, =u64=
- Zig: =i8= (two's complement, defined wrapping)
- Ruby: arbitrary precision (no fixed width)
- JavaScript: =Number= (IEEE 754 double — -1 is exact, but 0 has ±0 ambiguity)

The GF(3) trit {-1, 0, +1} must cross every one of these boundaries *identically*.

** The Sign-Magnitude Encoding (Mantissa/Significand Split)

#+begin_src text
  ┌──────────────────────────────────────────────────────┐
  │  TRIT WIRE FORMAT (FFI-safe, 2 bytes)                │
  │                                                      │
  │  Byte 0: SIGN (the "mantissa" = polarity)            │
  │    0x00 = ERGODIC  (zero, coordinator, boundary)     │
  │    0x01 = PLUS     (+1, generator, warm)             │
  │    0x02 = MINUS    (-1, validator, cold)             │
  │                                                      │
  │  Byte 1: SIGNIFICAND (the color self-address)        │
  │    0x00..0xFF = hue sector (hue * 256 / 360)         │
  │                                                      │
  │  Never use signed integers across FFI.               │
  │  The sign is ALWAYS byte 0, ALWAYS unsigned.         │
  │  The significand is ALWAYS byte 1, ALWAYS unsigned.  │
  └──────────────────────────────────────────────────────┘
#+end_src

Why 0x02 for MINUS instead of 0xFF?  Because 0xFF is *exactly* the trap:
two's complement -1 in any width.  The treacherous indeterminism is that
a C =int8_t= value of -1 cast to =uint8_t= gives 0xFF, which is *not*
semantically -1 in our encoding.  We dodge the trap entirely by never
using signed wire values.

** Self-Similarity: Golden Angle Separation

SplitMix64 generates hues separated by the golden angle (137.508°).
Consecutive hues are maximally dispersed.  But every *other* hue
differs by 2 × 137.508° = 275.016° ≡ -84.984° (mod 360°).

The sequence is *self-similar at worst every other interaction*:
the subsequence of even-indexed hues and odd-indexed hues each form
their own golden-angle-separated series, shifted by 137.508°.

This is the *Fibonacci lattice* on the circle: the densest packing
of hues such that no two are perceptually close, and the pattern
at scale 2n is a rotated copy of the pattern at scale n.

;; --- Signed Trit (FFI-safe wire format) ---
;; Never use signed integers across FFI boundaries.
;; The sign is always byte 0, always unsigned.

(def SIGN-ERGODIC 0x00)
(def SIGN-PLUS    0x01)
(def SIGN-MINUS   0x02)

(defn trit->wire
  "Encode a trit {-1, 0, +1} as FFI-safe [sign significand] pair."
  [trit hue]
  (let [sign (case (int trit)
               -1 SIGN-MINUS
                0 SIGN-ERGODIC
                1 SIGN-PLUS)
        significand (int (* (/ hue 360.0) 256))]
    {:sign sign
     :significand (bit-and significand 0xFF)
     :bytes [(unchecked-byte sign) (unchecked-byte significand)]}))

(defn wire->trit
  "Decode FFI-safe wire bytes back to trit + hue."
  [{:keys [sign significand]}]
  (let [trit (case (int sign)
               0x00  0
               0x01  1
               0x02 -1)
        hue (* (/ significand 256.0) 360.0)]
    {:trit trit :hue hue}))

;; --- SplitMix64 (deterministic color from identifier) ---

(def GOLDEN-GAMMA 0x9e3779b97f4a7c15)
(def MASK64 (unchecked-dec (bit-shift-left 1 64)))

(defn mask64 [n] (bit-and n MASK64))

(defn splitmix64 [seed]
  (let [z (mask64 (unchecked-add seed GOLDEN-GAMMA))
        z (mask64 (unchecked-multiply (bit-xor z (unsigned-bit-shift-right z 30))
                                       0xbf58476d1ce4e5b9))
        z (mask64 (unchecked-multiply (bit-xor z (unsigned-bit-shift-right z 27))
                                       0x94d049bb133111eb))]
    (bit-xor z (unsigned-bit-shift-right z 31))))

(defn state->hue [value]
  (mod (* (bit-and (unsigned-bit-shift-right value 16) 0xFFFF)
          137.508)
       360.0))

(defn hue->trit [hue]
  (cond
    (< hue 120) 1
    (< hue 240) 0
    :else -1))

   :minus   {:trit -1 :name "SECRET"       :access :holder-only}})

;; --- Arena (Diegetic Open Game) ---

(defn play
  "Forward pass: resolve identity sources → unified document."
  [sources]
  (let [{:keys [did-gay did-plc did-ens world-id]} sources]
    {:resolution {:did-gay did-gay
                  :did-plc (or did-plc (did-gay->did-plc did-gay))
                  :did-ens did-ens
                  :world-id world-id}
     :step :play}))

(defn coplay
  "Backward pass: verification attestations flow back."
  [resolution votes]
  (let [trits (map :trit votes)
        sum (reduce + trits)
        balanced? (zero? (mod sum 3))]
    {:attestations votes
     :gf3-sum sum
     :balanced? balanced?
     :consensus? (and balanced? (>= (count (filter :approve votes)) 2))
     :step :coplay}))

(defn arena-round
  "One round of the diegetic open game."
  [sources]
  (let [resolved (play sources)
        ;; Three agents vote independently (vibesnipe pattern)
        alice-vote   {:agent "ALICE"   :trit -1 :approve true
                      :check :world-id-fresh}
        arbiter-vote {:agent "ARBITER" :trit  0 :approve true
                      :check :gf3-balanced}
        bob-vote     {:agent "BOB"     :trit  1 :approve true
                      :check :document-valid}
        result (coplay resolved [alice-vote arbiter-vote bob-vote])]
    (merge resolved result)))

  F: Arena → GF(3), F(f ⊗ g) = F(f) + F(g), F(f >> g) = F(f) + F(g)
